# Antigravity IDE System Rules: Frontend Parity & Systematic Changes

## 1. Universal Feature Implementation
* **Enforce Parity:** When instructed to add, modify, or remove a frontend feature, you must apply the changes across **all** existing frontend versions in the workspace (e.g., web, mobile apps, admin panels).
* **No Partial Updates:** Do not generate or modify code for a single frontend environment if equivalent environments exist. Automatically mirror the implementation across all relevant interfaces.

## 2. Universal Systematic Changes
* **Global Refactoring:** When making systematic, architectural, or "under-the-hood" changes (e.g., updating API payload structures, changing state management logic, modifying shared utilities, or altering design system tokens), you must apply these changes universally across the entire project ecosystem.
* **Consistency Checks:** Ensure that shared logic and architectural modifications do not break, ignore, or leave behind legacy implementations in any frontend platform. Maintain strict synchronization.

## 3. Web change paraty
* If you are editing web interface you must make it very user friendly for both desktop and mobile
